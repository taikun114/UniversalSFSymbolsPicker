# UniversalSFSymbolsPicker

This is a Swift package that allows you to implement a highly customizable SF Symbols picker available across a wide range of platforms.

> [!NOTE]
> This package was created with the help of generative AI. Please be aware that it may contain writing styles that do not follow best practices or other potential issues.
>
> This package is a beta version currently under development.\
Since it has not yet been thoroughly tested, it may not function properly on some platforms or may contain bugs.\
> However, I have confirmed that it generally works on platforms other than watchOS.

## Supported Platforms

- macOS 14.0 or later
- iOS / iPadOS 17.0 or later
- tvOS 17.0 or later
- watchOS 10.0 or later (currently not working but planned for future support)
- visionOS 1.0 or later

## How to Use

The documentation is currently under preparation. I will update it once development has progressed further.

A demo app for trying out this package's functionality is located in `Examples/USSPDemo`, so please check it out if you'd like to start using the package right away.

## Why did I release it even though it’s still in beta?

I released this package even though it’s not yet complete because I wanted to implement it in an update for “[Mocolamma](https://github.com/taikun114/Mocolamma),” the app I’m developing, and release it right away.\
I had been holding off on releasing an update for Mocolamma because I wanted to include the symbol picker in it, and I wanted to get this package to a point where it worked reasonably well, release the Mocolamma update, and then continue development.\
While I could have implemented the package under development as a local package in Mocolamma and released it until this package was finished, Mocolamma is an open-source app and I wanted to make the source code public since this package is also open-source. Therefore, I decided to release it even though it’s still incomplete.

For this reason, while you may incorporate it into the app you are developing if you understand the risk that it may not function properly, I do not recommend doing so until it is officially released as version 1.0.0.\
Even so, I'd love to hear your feedback on what you've noticed while actually using it, so please feel free to open an issue if you have any questions or encounter any problems!
