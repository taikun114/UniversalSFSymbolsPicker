# AGENTS

## プロジェクトについて

プロジェクト名: **UniversalSFSymbolsPicker**

ターゲットプラットフォームは次の通り:

- macOS 14.0またはそれ以降
- iOS / iPadOS 17.0またはそれ以降
- tvOS 17.0またはそれ以降
- watchOS 10.0またはそれ以降（現在は非対応だが将来的に対応予定）
- visionOS 1.0またはそれ以降

### プロジェクト概要

UniversalSFSymbolsPickerはSwiftUIで利用可能な高機能・マルチプラットフォーム対応・高いカスタマイズ性のSF Symbolsピッカーを提供するSwiftパッケージです。\
UniversalSFSymbolsPickerパッケージ自体と、パッケージの細かい動作を確認したり検証したりするための`USSPDemo`というデモアプリによる2構成です。

## 開発方針

### コメントについて

UniversalSFSymbolsPickerの開発においては、**コード内に記載するコメントは英語で記載する必要**があります。

#### 言語・ローカライズに関して

UIに表示される言語やローカライズ関する注意点は次の通り。

- デフォルトのアプリ・パッケージ言語は英語で、複数の言語にローカライズを行ってマルチ言語対応を行う
  - デフォルトのUIテキストやローカライズキーは英語で記載すること
- ローカライズにはString Catalogシステムを使用する（通常の`Text('text')`のようなテキストはローカライズキーとして認識される）
  - ただし、パッケージ側のテキストは確実にモジュール側のローカライズを使用できるように、`String(localized: "text", bundle: .module)`形式（またはそれに相当する形式）で記述する必要があります。
- ユーザーに表示されるエラーメッセージやアラートなどに表示されるエラーメッセージもきちんと翻訳されるように実装する
  - ただし、`print()`文で出力されるデバッグログやコンソール出力はローカライズせず、英語で出力されるようにする必要がある

## プロジェクトのメンテナンスについて

ユーザーから次のようなことを言われた場合は、メンテナンス情報が記載されている`AGENTS/Maintenance.md`を読み、対処してください。

- 新しいバージョンのSF Symbolsに対応して
- ロケールフィルタリングがきちんと動作しているか確認して
- パッケージのローカライズに新しい言語を追加して

また、ローカライズに関する作業を行う場合は先に`AGENTS/Maintenance.md`を読んでから作業を行ってください。
